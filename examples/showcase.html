Design Tokens

Two-tier token architecture. Raw tokens hold literal values; semantic tokens alias them by purpose. Components only consume semantic tokens — never raw values or hex codes.

Semantic Color Tokens
Token Value (resolved) Purpose
/* Raw tokens — literal values */ --pos-raw-color-blue-600: #228be6; --pos-raw-color-neutral-0: #ffffff; /* Semantic tokens — reference raw via var() */ --pos-color-accent: var(--pos-raw-color-blue-600); --pos-color-bg: var(--pos-raw-color-neutral-0);
Theming

Switch themes by setting data-pos-theme on any ancestor element. CSS custom properties cascade through Shadow DOM boundaries automatically — components require no changes.

data-pos-theme="light"
Primary action Secondary Ghost Danger
/* Default theme — set in dist/tokens/theme.css */ :root, [data-pos-theme="light"] { --pos-color-accent: var(--pos-raw-color-blue-600); } /* Tenant override — override semantic tokens only */ [data-pos-theme="tenant-acme"] { --pos-color-accent: #e63946; --pos-color-accent-hover: #c1121f; }

Plugin Runtime

Plugins are dynamically imported ES modules that register a Custom Element. The host injects a hostSDK before the element connects — giving the plugin access to getToken() and emit().

Live plugin — plugin-hello
Loading plugin…
import { loadPlugin, createHostSDK } from '/dist/pos-design-system.js'; // 1. Load and register the custom element const { tagName } = await loadPlugin({ url: '/examples/plugins/hello-plugin.js', tagName: 'plugin-hello', }); // 2. Create element, inject SDK before connecting to DOM const el = document.createElement(tagName); el.hostSDK = createHostSDK(el); // { emit, getToken } document.body.appendChild(el); // Inside the plugin, SDK is available in connectedCallback: // this.hostSDK.getToken('--pos-color-accent') → '#228be6' // this.hostSDK.emit('plugin:hello:action', { ... })
