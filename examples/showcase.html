Design Tokens

Two-tier token architecture. Raw tokens hold literal values; semantic tokens alias them by purpose. Components only consume semantic tokens.

Semantic Color Tokens
TokenValuePurpose
/* Raw tokens — literal values */ --pos-raw-color-blue-500: #0067DD; --pos-raw-color-neutral-0: #FFFFFF; /* Semantic tokens — reference raw via var() */ --pos-color-action-primary: var(--pos-raw-color-blue-500); --pos-color-background-primary: var(--pos-raw-color-neutral-0);
Theming

Switch themes by setting data-pos-theme on any ancestor. CSS custom properties cascade through Shadow DOM automatically — components require no changes.

data-pos-theme="light"
Primary Outline Ghost Danger
Badge

Plugin Runtime

Plugins are dynamically imported ES modules that register a Custom Element. The host injects a hostSDK before the element connects — giving the plugin access to getToken() and emit().
